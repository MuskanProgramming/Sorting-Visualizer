Bubble Sort

Repeatedly steps through the list, compares adjacent elements, and swaps them if they are in the wrong order.
  • Best Time Complexity : O(n)
  • Average Time Complexity : O(n²)
  • Worst Time Complexity : O(n²)
  • Space Complexity : O(1)
  • Method Used : Comparison-based
  • Stability : Stable

Comb Sort

An improvement of bubble sort, it uses a gap sequence to eliminate small values near the end of the list.
  • Best Time Complexity : O(n log n)
  • Average Time Complexity : O(n² / log n)
  • Worst Time Complexity : O(n²)
  • Space Complexity : O(1)
  • Method Used : Comparison-based
  • Stability : Unstable

Heap Sort

Converts the array into a heap data structure and repeatedly extracts the maximum element.
  • Best Time Complexity : O(n log n)
  • Average Time Complexity : O(n log n)
  • Worst Time Complexity : O(n log n)
  • Space Complexity : O(1)
  • Method Used : Heap-based
  • Stability : Unstable

Insertion Sort

Builds the sorted array one item at a time by comparing each element with the previous elements and inserting it in the correct position.
  • Best Time Complexity : O(n)
  • Average Time Complexity : O(n²)
  • Worst Time Complexity : O(n²)
  • Space Complexity : O(1)
  • Method Used : Comparison-based
  • Stability : Stable

Selection Sort

Repeatedly selects the smallest (or largest) element and swaps it with the element in the correct position.
  • Best Time Complexity : O(n²)
  • Average Time Complexity : O(n²)
  • Worst Time Complexity : O(n²)
  • Space Complexity : O(1)
  • Method Used : Comparison-based
  • Stability : Unstable

Shell Sort

An optimized version of insertion sort that allows the exchange of far-apart elements to improve sorting speed.
  • Best Time Complexity : O(n log n)
  • Average Time Complexity : O(n^(1.25))
  • Worst Time Complexity : O(n²)
  • Space Complexity : O(1)
  • Method Used : Comparison-based
  • Stability : Unstable
